Работай по Documentation First Development. Сначала изучи проект, если он есть, задай вопросы через ask-before-assuming. Если что-то не ясно, не чётко, то нужно задавать вопросы. Затем создай документацию в docs/. Если проект большой, разбей документацию по модулям. После этого создай docs/development-pipeline.md с этапами реализации, проверками и критериями готовности.
