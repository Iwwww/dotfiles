Задокументируй модуль проекта.

Требования:

- Вся документация только на русском языке.
- Писать только в `docs/`, исходники не менять.
- Сохранить структуру модуля внутри `docs/`.
- Для каждого файла модуля создать отдельный `.md`.
- Создать общий `README.md` по модулю.
  Этап 1:
- Построй карту модуля.
- Определи порядок чтения файлов.
- Если уже есть подмодули с документацией, то читай документацию. Если будут неясности по документации, то читай исходники и дополняй документацию.
- Делегируй разбор каждого файла агентам параллельно.
- Каждый агент должен дать краткую выжимку и записать doc по своему файлу.
- Для каждого файла опиши: назначение, основные функции/макросы/структуры, зависимости, связи с другими файлами, роль в модуле.
- Если файлы однотипные, маленькие и лежат в одной директории и их есть смысл документировать в одном документе, то так и делай через одного агента.
- Если файлы не являются кодом, а, например, конфигом или медиа файлами, то также документировать в одном документе эту группу через одного агента.
  Этап 2:
- На основе собранного материала напиши общую документацию по модулю.
- Обязательно опиши: назначение модуля, структуру файлов, общий flow работы, данные и зависимости, взаимодействие с другими частями системы, влияющие настройки и режимы, точки расширения, типовые изменения, ограничения и риски.
  Этап 3:
- Сделай короткое агентское ревью документации.
- Если есть замечания по точности, полноте или понятности, исправь их.
  Результат:
- per-file документация в `docs/` на русском,
- `README.md` модуля на русском,
- краткий отчет: какие файлы созданы, что нашли на ревью, что исправлено.
