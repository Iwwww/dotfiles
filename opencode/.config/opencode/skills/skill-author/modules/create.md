# Create

Создание нового opencode skill.

## Workflow

### 1. Анализ

Прочитай целевую конфигурацию/код. Пойми:
- Что должен делать скилл
- Какие правила/знания нужно кодифицировать
- Есть ли уже AGENTS.md или инструкции — что оттуда переезжает

### 2. Вопросы пользователю

Прежде чем писать код, задай вопросы:
- Куда положить скилл: `~/.config/opencode/skills/` (глобальный) или `.agents/skills/` внутри проекта?
- Один файл или модульный роутер? (роутер если >2 интентов)
- Авто-триггер или ручной?
- Какие модули нужны?
- Язык инструкций (русский/английский)?

### 3. Определить расположение

Спросить у пользователя. Доступные варианты:
- `~/.config/opencode/skills/<skill-name>/` — глобальный (доступен всем проектам)
- `<project>/.agents/skills/<skill-name>/` — project-local

### 4. Структура

```
<location>/<skill-name>/
├── SKILL.md
└── modules/
    ├── module1.md
    └── module2.md
```

### 5. SKILL.md

```yaml
---
name: <kebab-case>
description: >
  One paragraph. Clear trigger conditions.
  Use when ...
---
```

Секции:
- Router (если модульный): таблица интентов
- Базовые правила: что всегда применимо

### 6. Модули

Правила:
- Один файл = один concern
- Max ~60 строк
- Без воды. Каждая строка должна оправдывать своё существование

### 7. Интеграция

- Скилл авто-дискаверится из `~/.config/opencode/skills/` (глобальные) или `<project>/.agents/skills/` (project-local)
- Не добавлять в opencode.json — не требуется
- Опционально: если проект использует AGENTS.md — добавить ссылку на скилл
