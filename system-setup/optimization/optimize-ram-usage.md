### 1. Проблема
Система испытывала высокий Load Average (около 4-5) и микрофризы, несмотря на низкую загрузку процессора (около 14%). Главной причиной стала острая нехватка оперативной памяти (доступно 8 ГБ) из-за архитектуры браузера Zen, который создавал множество тяжелых изолированных процессов для вкладок. Из-за этого начался жесткий "swap thrashing": быстрый ZRAM (4 ГБ) переполнился, и ядро начало активно гонять гигабайты памяти в медленный swap-файл на SSD, создавая узкое место в I/O (вводе-выводе). Ситуацию усугубляли неоптимальные параметры ядра для кэша, конфликтующие опции монтирования файловой системы Btrfs и отсутствие аппаратного ускорения видео. Своевременное обслуживание и оптимизация ОС критически важны для стабильной работы Arch Linux. [wiki.archlinux](https://wiki.archlinux.org/title/System_maintenance)

### 2. Решение с краткими комментариями в коде

```bash
# --- 1. Настройка гибридного Swap (ZRAM + Файл на SSD) ---
# Файл: /etc/systemd/zram-generator.conf
[zram0]
zram-size = ram                  # Выделяем до 100% ОЗУ (8 ГБ) под сжатый быстрый кэш
compression-algorithm = zstd     # Используем эффективный алгоритм сжатия
swap-priority = 100              # ZRAM заполняется первым, SSD-файл (pri=0) остается для гибернации
fs-type = swap

# --- 2. Оптимизация ядра (Sysctl) ---
# Файл: /etc/sysctl.d/99-zram.conf и 99-io.conf
vm.swappiness = 180              # Агрессивно сжимаем анонимную память в ZRAM, защищая файловый кэш
vm.page-cluster = 0              # Отключаем чтение с упреждением (read-ahead) — для ZRAM это не нужно
vm.watermark_boost_factor = 0    # Отключаем резкие всплески агрессивной работы kswapd
vm.watermark_scale_factor = 125  # Делаем освобождение памяти ядром более плавным
vm.vfs_cache_pressure = 50       # Дольше удерживаем в ОЗУ кэш файловой системы (dentry/inodes)
vm.dirty_background_ratio = 5    # Начинаем фоновую запись на диск при 5% грязных страниц в памяти
vm.dirty_ratio = 10              # Принудительная запись при 10% (защита от гигантских I/O фризов)

# --- 3. Настройка дисковой подсистемы (Btrfs) ---
# Файл: /etc/fstab
# Удалили конфликтующий 'relatime', оставили чистый 'noatime' для отключения записи времени доступа
UUID=... / btrfs rw,noatime,compress=zstd:3,ssd,discard=async,space_cache=v2,subvol=/@ 0 0

# Отключаем fstrim, так как Btrfs с параметром discard=async очищает SSD на лету без фризов
sudo systemctl disable --now fstrim.timer

# --- 4. Настройка сети (BBR) ---
# Файл: /etc/modules-load.d/bbr.conf
tcp_bbr                          # Загружаем модуль современного контроля перегрузки TCP при старте

# --- 5. Аппаратное ускорение графики (Intel iGPU) ---
# Устанавливаем драйверы для снятия нагрузки декодирования видео с процессора
sudo pacman -S intel-media-driver libva-utils

# --- 6. Оптимизация браузера Zen (через about:config) ---
# dom.ipc.processCount.webIsolated = 1 или 2  (Радикально снижаем потребление ОЗУ, группируя сайты)
# browser.sessionstore.interval = 300000      (Сохраняем сессию раз в 5 минут для защиты SSD от износа)
# media.ffmpeg.vaapi.enabled = true           (Включаем аппаратное декодирование видео)
# gfx.webrender.all = true                    (Принудительно включаем отрисовку интерфейса через GPU)
```
