# Agent workflow

## Docs-first rule

Перед нетривиальными изменениями сверься с официальной документацией Eww:
- синтаксис конфигов;
- опции `defwindow`;
- встроенные виджеты;
- expression language;
- магические переменные;
- GTK CSS / ограничения темизации;
- CLI-команды установленной версии.

Не выдумывай свойства, атрибуты, флаги или имена переходов.

При расхождении локальной версии с документацией — проверяй локально:

```sh
eww --help
eww open --help
eww close --help
eww update --help
eww reload --help
```

Если в проекте уже есть рабочий Eww-код, сохраняй версионные паттерны, если они не сломаны.

## Порядок действий перед редактированием

1. Прочитай существующие файлы: `eww.yuck`, импорты `.yuck`, `eww.scss`, импорты стилей, скрипты.
2. Определи: Wayland, X11 или оба.
3. Определи композитор (River, Sway, Hyprland, Niri, etc).
4. Проверь, использует ли код команды композитора.
5. Сделай минимальный безопасный патч.
6. Запусти или предложи команды валидации.
7. Если валидация невозможна — скажи об этом.

Не переписывай весь конфиг без явной просьбы.
